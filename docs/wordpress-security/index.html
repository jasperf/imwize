Updates and Backups

So often clients neglec to update their site or forget to make backups

Backup Database and Files

Made a site change, added a blog post? Backup! Not just on the server, but add a third location if possible.

And yes, do update and update often whethere it is themes, core or plugins.

Backup on Server and Third Location

Make sure you backup on the server using a plugin or script that creates a unique folder and zipped package like BackupBuddy. BB also allows the automatic storing of packages on third locations like AMZ or DropBox.

Backup Locally

Always good to have a local storage of your precious site too of course.

Server Tips

Server Security needs depend on your setup. When you run a dedicated server you will take of all, when on a VPS you do quite a bit and on Shared almost zero. But there are still things that you can do

Use SFTP, SSL and SSH when Possible

Do NOT use FTP as it is insecure. Use SFTP or Shell. Also do use SSL so connection to server is encrypted and identity site secured.

Check File and Directory Permissions

Make sure all files and folders have the correct permissions. Most of the time is should bechmod 644 for files and chmod 755 for folders

Clean Up Outdated Apps

If you run staging areas on your server or test apps, or version controlled versions, make sure they also stay updated, or better yet remove them!

Secure WordPress Setup

Database Prefix, No README, No Online Editing

Do not use wp_ as a prefix. Use a unique prefix. Harder to hack using MySQL injections that way.

Remove the README and About files. Not needed and they give away info.
