Introduction

Introduction

Welcome to this WordPress SEO Meetup. Great for you to have joined in.

This meetup will be about WordPress SEO where we explain how you too can improve your WP site's ranking by using simple tweaks and tools out there.

Content and Layout

It will have to be told time and again to clients. There is NO SEO Voodoo. Most of the work will go into good content and content based on a case study on what keywords you would like to be found

Content

Good, relevant up-to-date content based on a keyword strategy and often added new content is something that users love and Google loves. This to become an authority in your field

Try to add new content through a blog or (micro) blogging elsewhere and refering to the main site. Google Keyword Planner, Quora, forums related to your site's product or services and Yoast WordPress SEO are great tools

Enough content is also vital. Three hunderd or more words per article and plenty of articles!

Read this! and use Jetpack Statistics or Google Analytics to keep track of progress

Viewport | Above the Fold

Layout of your site is important as well. You need to think about things such as above the fold content, benefits of a slider, use of dropdowns and so on. Content that is out of sight will not be read. The wrong distractions will lead visitors away from what you wanted them to visit you in the first place.

With Chrome Page Analytics and being logged in to GA you can analyse pages and see how each element does.

Reponsive Design

Nowadays Google Search will indicate if your site is mobile friendly meaning whether your site is responsive or at least has a mobile friendly display. Here more news on this topic. Use great tools out there that apply media queries such as Bootstrap or CSS Skeleton. No need to reinvent the wheel. Checking whether your site is responsive can be done using Firebug, Google PageSpeed or many other tools.

Backlinks

Definition: an incoming hyperlink from one web page to another website. There are several plugins that can help you get backlinks. Several are useful in publishing your blog posts on social media. Social Networks Auto Poster is one. But of course it great when real people talk about you on the web. See also Google's explanation and WP Beginner's Post on Social Media Plugins.



Site Semantics

Headers, lists,..

Although it not something that you will think of right away, the proper formatting of your site in headers, lists and paragraphs will help you keep your readers on your site and is also appreciated by search engines.

Headers

Make sure you have at least H1-H3 header tags. Normally H1 is reserved for the title of your site and or logo and the H2 for page titles. H3 normally add for sub heading. And if you need more there is always H4-6 and CSS to create variations

Alt Tags

Alt tagging your images will help search engines understand what the image is about. And although Google is working on getting better at auto captioning images this is still relevant for our sites.

Meta Description

Meta description is still a valid snippet of information used by search engines and shown on search result pages (SERPs). Yoast's WordPress SEO plugin has this included and will tell you whether the description is too long or not. It should be between 150 and 160 characters long. See Moz on Meta Description here.

W3C Validator and Webmaster Tools

One way to test your code and see that all is good, no closing tags missing and proper formatting applied is checking your code at W3C. There is a checker for CSS as well as HTML and it is for free! The other free tool is Google Webmaster Tools / Bing Webmaster Tools

Compression and Browser Caching

To load your site as quickly as possible all images should be compressed well and the site content should be gzipped by the server. Many of the techniques here are tips for the browser how to deal with loading of your site data. The "FarFutureExpiry" Plugin can help with Expires Header tweaks

Most websites these days load many JavaScripts and or CSS files. This slows down your site. To speed up loading you can concatenate and minify scripts.. There are plugins for this like one mentioned below. More on conc. and min. a little later

Server Response Time

The quicker your server responds or loads the site the better. So a low TTFB is essential. Quick check from the command line 

curl -o /dev/null -w "Connect: %{time_connect} TTFB: %{time_starttransfer} Total time: %{time_total} \n" http://site.com

Google would like it to be less than 200 ms.

To server your content just as quickly to customers in Norway as in Singapore your site needs to be locally everywhere in the world. This is what you can do with a CDN. It serves static content from different locations globally to server client content quicker. I use MacxCDN and am very happy with them.

Image Compression | Size

Another issue a lot of sites have is that they either resize images in the browser or use images that are rather larger. There are command line and online tools you can install with Homebrew or any other package manager. Commands with jpegoptim or optipng such as:

jpegoptim --max=90 --strip-all --preserve --total image.jpg

or 

optipng -o7 long-logo.png

to deal with this as well as plugins such as EWWW optimizer.

Responsive and Adaptive Images

And then there is the issue of serving the right image for the right device. Imagewize is looking into bringing the future here with PictureFill or HISRC as solutions to the multi device and bandwidth environments. Server Side Caching

Many of you might be familiar with browser cache. Also see compression point discussed earlier. Not that many might know server caching techniques and or proxy servers. For Server Side Caching MemCache or Varnish are good options. PHP has built in OpCache in 5.5.+ and should take over APC. A plugin to help out would be WP Super Cache or W3 Total Cache.

Concatenation and Minification Extended

Concatenating just means putting several files together in one large file. This speeds up the loading of a site as http protocol can only download x files per time . Minification or uglification is compressing your files code removing all empty lines and putting all code on one line, making the file smaller. Grunt is a JScript Task Runners that can help automate this
